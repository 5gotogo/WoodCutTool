# WoodCutTool SEO Strategy & Implementation Log

This doc records (1) the diagnosis of the site's SEO, (2) what was implemented in code
for priorities P2–P3, and (3) the operational P4 actions that cannot be done in code and
need a human (backlinks, E-E-A-T, content focus).

---

| Priority | Issue | Status |
|----------|-------|--------|
| **P1** | **Topic dilution** — ~25 of 60 blog posts and 30+ tools-hub apps are unrelated to woodworking (QuiltFit, Tinnitus, Cadenza, PDF scanner…). Dilutes topical authority for woodworking keywords. | ⏳ **Needs owner decision** (see below) |

## P2 — Technical SEO (implemented)

All generated by build scripts so they stay consistent and survive rebuilds.

### Open Graph + Twitter Cards — site-wide
- New shared module [`scripts/seo-meta.mjs`](../scripts/seo-meta.mjs) emits `og:*` and
  `twitter:*` tags from each page's title/description/canonical.
- Wired into every generator: blog, learn, apps, and the hand-written pages
  (via [`scripts/apply-seo-meta.mjs`](../scripts/apply-seo-meta.mjs)).
- Blog + learn articles use `og:type=article`; everything else `website`.

### BreadcrumbList structured data
- Added to learn articles, learn landing pages, blog posts (in their generators) and to
  hand-written tool/template pages (via `apply-seo-meta.mjs`, injected after `<body>` so
  it never collides with the SoftwareApplication JSON-LD that `apply-tool-schemas.mjs`
  replaces). Coverage went from 1 page → 80+ pages.
- Gives Google the breadcrumb trail it can show in results (CTR lift).

### HowTo structured data
- Added to the 6 calculators (plywood, cut-list, wood-waste, board-foot, stair-stringer,
  tile) in [`scripts/apply-tool-schemas.mjs`](../scripts/apply-tool-schemas.mjs) via a new
  `howto` field + `howToSchema()`. Each is a 4-step "how to use this calculator" — eligible
  for HowTo rich results.

### Social share image
- [`assets/og/woodcuttool-og.png`](../assets/og/woodcuttool-og.png) — 1200×630, brand
  colors, generated from [`woodcuttool-og.svg`](../assets/og/woodcuttool-og.svg) (kept as
  the editable source). Referenced as the default `og:image` everywhere.
- **TODO (nice-to-have):** per-page OG images (e.g. one per calculator) would lift social
  CTR further. Current setup uses one shared image, which is fine to ship.

### Build order
`package.json` `build` now runs: apps → blog → learn → tool-schemas → **apply-seo-meta →
add-tool-guide-links** → sitemap → validate. `apply-seo-meta` runs *after* tool-schemas
on purpose (breadcrumb vs SoftwareApplication JSON-LD ordering).

## P1 — Topic dilution (needs an owner decision)

The single biggest structural lever. `woodcuttool.com` ranks for woodworking, but the
site also hosts a large unrelated app catalog. Pick one path:

- **A. Separate the unrelated apps** (recommended) — move non-woodworking app/blog content
  to a subdomain or separate site, or `noindex` + remove from the main sitemap. Makes
  woodcuttool.com ~100% woodworking and concentrates topical authority.
- **B. Internal-link isolation** — keep pages but split the tools hub into "Woodworking
  tools" and "Other apps", and stop linking the off-topic apps from the woodworking cluster.
- **C. Accept it** — if the app pages drive meaningful installs/revenue, keep as-is.

No code change made here — this is a strategy call. Once decided, it's straightforward to
implement (sitemap filtering, robots/meta, hub restructuring).

## P4 — Authority & E-E-A-T (operational — for a human)

These move rankings but can't be done in the repo. Rough priority order:

1. **Earn backlinks from woodworking communities.**
   - Participate genuinely on r/woodworking, woodworking forums, Facebook groups; link the
     free calculators when actually relevant (don't spam).
   - Submit to tool directories / "free woodworking tools" roundups.
   - Reach out to woodworking bloggers/YouTubers — a free, no-login plywood optimizer is an
     easy mention.

2. **Build E-E-A-T signals.**
   - Add a real **About** page: who builds this, woodworking background, why trust the math.
   - Add author bylines + short bios on learn/blog articles.
   - Show the methodology behind the calculators (how kerf/waste are computed) — first-hand
     expertise is what Google's "Experience" axis rewards.

3. **Publish one citable, data-driven asset.**
   - e.g. "We analyzed N cabinet cut plans — average plywood waste was X%." Original data
     attracts links far better than another how-to.

4. **Refocus content production on woodworking** (ties to P1) — stop splitting blog effort
   across unrelated apps; double down on the cutlist/plywood/cabinet topics that already
   perform.

5. **Monitor & maintain.**
   - Google Search Console: watch impressions/CTR per page; iterate titles where CTR lags
     (the title A/B variants are in [`docs/seo-titles-descriptions.md`](seo-titles-descriptions.md)).
   - Validate structured data in Google's Rich Results Test after deploy.
   - Keep Core Web Vitals healthy: the site is already lean (homepage ~17 KB, external CSS,
     no images on tool pages) — **don't add heavy JS or large hero images.**
